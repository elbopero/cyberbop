Cracking hashes with John the Ripper

In this walkthrough, we’ll use John the Ripper, a popular and powerful open-source password cracking tool used in cybersecurity to test password strength and discover weak passwords. John the Ripper is highly versatile and supports a variety of password hash types, making it a go-to tool for security professionals and ethical hackers. For this demonstration, we’ll create our own hash and crack it to reveal the password. By leveraging John the Ripper, we can identify weak passwords, understand how attackers exploit password vulnerabilities, and learn best practices for strengthening authentication measures. We’ll go through the process of cracking hashes step-by-step to illustrate how attackers might approach password attacks and how to safeguard systems against these vulnerabilities.

Introduction

To perform the attack, we will be using the password cracking tool John the Ripper which comes preinstalled with Kali Linux. We will be creating our own hash, identify the hash and then crack it to reveal the password.


Creating our own hash

To create our hash im going to use CyberChef which can be found at https://gchq.github.io/CyberChef/ and I will use the MD5 hashing algorithm in this example. I will enter "elephant" as input to the function, and this generates the hash "e4b48fd541b3dcb99cababc87c2ee88f".


In step 1 we choose "hashing" from the left side menu. In step 2 we drag MD5 (the algorithm we are using) into the recipe field. In step 3 we enter our text that we want to create the hash for (elephant in this example). In step 4 we receive the output hash from the function: e4b48fd541b3dcb99cababc87c2ee88f, this is the hash we will be cracking.

John the Ripper

To use John the Ripper we need to find out what type of hash format we are dealing with, we created it so we know it is MD5, but lets assume we didn't know (like real attackers). We also need to provide a file containing different passwords, that John the Ripper will use to calculate hashes for and compare it against the hash we are trying to crack.

We will start with "john" to specify which tool we are using, and then choose which format, and with wordlist we will provide our file loaded with passwords (rockyou.txt) and lastly we will type our file that contains our hash that we want to crack.

Identifying hash format

In this part we will identify what type of hashing format we are dealing with. I just googled "Hash Analyzer" and chose my top pick, and entered the hash value into its analyzer field.

In the picture above we can see the analyzer tool identifies our hash as MD5 or MD4.

Cracking the hash

After typing "john --list-formats | grep -i "md5", all the MD5 formats that John supports are listed. In this we wouldn't necessarily be sure about which one it would be, or if it was a MD4 like the analyzer also suggested, and here my tip is to do the dirty work and try each one.

The password is cracked using the format raw-md5, and luckily for us this mean that the rockyou.txt file we used contains the md5 hash for the word "elephant", and John the Ripper has a match on the hash we gave it from our hash.txt file.
