Finding vulnerabilities with Nmap

In this walkthrough, we’ll use Nmap (Network Mapper) to scan for open ports and identify potential vulnerabilities on a target machine. Nmap is a powerful, open-source network scanning tool commonly used in cybersecurity for network discovery. We’ll be conducting this scan on Metasploitable 2, a purposefully vulnerable machine designed for learning and testing. By using Nmap, we can uncover critical information about exposed services, detect potential weaknesses, and learn how attackers might target systems. We’ll demonstrate various scanning techniques to get a detailed understanding of the target’s network setup and any exploitable vulnerabilities.

Introduction

To perform the scan, we will be using the Nmap tool which comes preinstalled with Kali Linux. Our attack target is going to be Metasploitable 2. In the next part (How to use Nmap) we will go through how we can use the tool.


How to use Nmap

When we are going to scan a device or an application (domain or a IP-address) we must have this in order to scan our target. Using Nmap is very easy and straightforward, we will simply write nmap in our terminal along with the IP-address of our attack target in this scenario. Figuring out the IP-address of our target can be done in multiple ways if we are on the same network, I will use Nmap in this case.

Different scan modes

Before starting to scan our target I want to let you know about the different types of scanning alternatives Nmap can perform. A basic scan will simply tell us what ports are open. Other types of scans are used to determine what services and version of the service is running, scans can be performed faster to save time but this makes it more detectable for our target. Scans can also be adjusted to look for specific known vulnerabilities. In the next part we will look at a basic scan.

Basic scan

To perform a basic scan we will just type in "sudo nmap target_ip".

In the picture above we can see the port numbers and what types of services are running on each port marked with red. Attackers seek to exploit vulnerable versions, this scan doesn't provide us this. In the next part we will look at how to find vulnerable versions during our scan.

Version scan

To perform a version scan we will add -sV to our command.

In the picture above we now see the "Version" columns showing which application and version (if identified) are running on each port. This is a crucial step in many cyberattacks, as there are many known vulnerabilities publicy known which attackers target, knowing which application and version of it is running lets attackers know what kind of vulnerabilities the target is exposed to.
